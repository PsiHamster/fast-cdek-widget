
## Проблема

Известные проблемы виджета:
1. При запуске подгружается >8000 пунктов выдачи заказов
2. При переинициализации старые обработчики событий продолжают реагировать, приводя к UB

## Решение 

- Парковка виджета в ДОМ страницы, переинициализация без пересоздания виджета.

- Расширение контракта апи передачей кода региона.

В Front/cdekWidgetSingleton.js реализован синглтон-блок виджета выбора ПВЗ

В usageSample перечислено как его можно использовать в приложении.



Для корректной работы надо научиться:

### 1. Давать пользователю выбрать регион. 

Подготовлен справочник cdekRegions.js, в котором есть связки - Название региона, Адрес вместе с столицей, код региона.

Пользователю можно дать селектор по региону, а в виджет прокидывать код + адрес.

Быстрых способов улучшить работу поиска внутри виджета не найдено, лучше закрывать пользователю карту, пока регион не выбран.

### 2. Бэкенд уже работает с получением списков офисов по регионам, и запросы должны сразу уменьшиться. 

В этой ветке в /backend/ добавлен только заголовок кэша, чтобы список офисов кэшировался на сутки.

UPD: КЭШИРОВАНИЕ ЧЕРЕЗ Cache-Control: public, max-age=86400 НЕ РАБОТАЕТ и приводит к тому, что при смене региона на первый запрос возвращаются первые значения. Нужно переписать php из официального репозитория на работу с e-tag.
И желательно также реализовать кэш на запросы к СДЭК, или предзагрузку всех офисов с дофильтрацией.
