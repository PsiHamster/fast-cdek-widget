
### Проблема

Известные проблемы виджета:
1. При запуске подгружается >8000 пунктов выдачи заказов
2. При переинициализации старые обработчики событий продолжают реагировать, приводя к UB

### Решение 

Парковка виджета в ДОМ страницы, переинициализация без пересоздания виджета.
Расширение контракта апи передачей кода региона.

В Front/cdekWidgetSingleton.js реализован синглтон-блок виджета выбора ПВЗ

В usageSample перечислено как его можно использовать в приложении.



Для корректной работы надо научиться:

1. Получать код региона из адреса

(Например, уметь искать город, для этого надо закэшировать на сервере два списка с бэка СДЭК - 

/location/cities?country_codes=RU&size={PageSize}&page={page}
/location/regions?country_codes=RU&size={PageSize}&page={page}

Давать искать по подстроке города, а затем возвращать регион, в котором этот город находится, и передавать в виджет address (адрес, составленный так, чтобы геолокатор яндекса смог точно определить город) + regionCode

Либо просто давать выбрать регион.

2. Бэкенд уже работает с получением списков офисов по регионам, и запросы должны сразу уменьшиться. 

В этой ветке добавлен только заголовок кэша, чтобы список офисов кэшировался на сутки.

Но желательно также реализовать кэш
